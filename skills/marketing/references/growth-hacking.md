# Growth Hacking

## Growth Loops

### Viral Loop
User invites → New user signs up → New user invites
- **Key metric:** Viral coefficient (K-factor). K > 1 = exponential growth
- **Example:** Dropbox referral ("Get 500MB free for each friend")
- **Design:** Make sharing effortless + reward both sides

### Content Loop
Create content → SEO traffic → Signups → User generates content → More SEO
- **Key metric:** Organic traffic growth MoM
- **Example:** Notion templates, Canva designs
- **Design:** User-generated content that ranks in search

### Paid Loop
Revenue → Reinvest in ads → New customers → More revenue
- **Key metric:** CAC payback period. Must be < LTV
- **Example:** Any product with positive unit economics
- **Design:** Optimize for shortest payback period

### Sales Loop
Customer success → Case study → Use in sales → New customers
- **Key metric:** Win rate with case studies vs without
- **Design:** Systematize case study creation

## Referral Program Design

### Two-Sided Incentive
| Trigger | Giver Gets | Receiver Gets | Example |
|---------|-----------|---------------|---------|
| Signup | Credit | Discount | "Give $10, Get $10" |
| Usage | Extended trial | Free month | "Invite 3, get free month" |
| Revenue | Commission | Discount | "Earn 20% recurring" |

### Implementation Checklist
- [ ] One-click sharing (pre-written message)
- [ ] Unique referral link per user
- [ ] Real-time tracking (user sees status)
- [ ] Instant reward delivery
- [ ] Built into product (not bolt-on)
- [ ] Prominent placement (dashboard, post-purchase)

### Prioritization (ICE Score)
- **I**mpact: How much will it move the metric? (1-10)
- **C**onfidence: How sure are we it'll work? (1-10)
- **E**ase: How easy to implement? (1-10)
- **ICE Score = I × C × E / 3**

Run experiments with highest ICE first.

## Product-Led Growth (PLG) Checklist
- [ ] Free tier or trial available (low friction entry)
- [ ] Self-serve onboarding (no sales call required)
- [ ] Time to value < 5 minutes
- [ ] In-product upgrade prompts (contextual, not annoying)
- [ ] Usage-based pricing (grows with customer)
- [ ] Built-in sharing/collaboration features
- [ ] Community-driven support
