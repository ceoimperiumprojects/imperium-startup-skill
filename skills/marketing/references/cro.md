# CRO (Conversion Rate Optimization)

## The CRO Process

1. **Measure:** Analytics + heatmaps + session recordings → find drop-off points
2. **Hypothesize:** "If we [change X], then [metric Y] will [improve Z] because [reason]"
3. **Test:** A/B test with statistical significance (>95% confidence)
4. **Learn:** Document results, win or lose
5. **Repeat:** Never stop optimizing

## High-Impact CRO Areas

### Signup Flow
- Remove unnecessary fields (every field reduces conversions 5-10%)
- Add social signup (Google, GitHub for dev tools)
- Show progress indicators
- Auto-fill where possible
- Mobile-first design
- Clear value prop above the form

### Pricing Page
- Lead with the recommended tier ("Most Popular")
- Use anchoring (show highest price first or use a decoy)
- Annual billing toggle with savings shown
- Feature comparison table
- FAQ section addressing common objections
- Social proof near CTA (logos, testimonials)
- Money-back guarantee

### Onboarding
- Time to first value < 5 minutes
- Progressive disclosure (don't overwhelm)
- Checklist with progress tracking
- In-app guidance (tooltips, walkthroughs)
- Welcome email sequence (see email-sequences.md)
- Identify and optimize the "aha moment"

### Landing Pages
- Message match (ad → landing page headline consistency)
- One CTA per page (don't dilute)
- Above-the-fold: Headline + subheadline + CTA + visual
- Social proof placement (near CTA)
- Remove navigation (reduce exits)
- Page load <3 seconds

### Forms
- Progressive profiling (ask more later, not upfront)
- Smart defaults (pre-select most common option)
- Inline validation (real-time error feedback)
- Clear labels (above the field, not inside)
- Optional vs required clearly marked

## Quick CRO Wins (No A/B Test Needed)
- Fix broken links and 404 pages
- Speed up page load time
- Make CTAs action-oriented ("Start free trial" not "Submit")
- Add social proof near conversion points
- Remove auto-playing videos
- Fix mobile layout issues
- Add search functionality
- Simplify navigation

## A/B Testing Rules
- Test one variable at a time
- Run until statistical significance (>95%)
- Minimum sample size: 1000 visitors per variation
- Run for at least 2 full weeks (capture weekly patterns)
- Don't peek at results early and call it
- Document EVERYTHING (even failed tests teach you)
