# Habit-Forming Products

The Hook Model for building products users can't put down. Distilled from Nir Eyal's "Hooked" (129 chunks in knowledge base).

## The Hook Model

Four phases that create user habits through repeated cycles:

```
    ┌──────────┐
    │ TRIGGER  │ ← What prompts the user to act?
    └────┬─────┘
         ▼
    ┌──────────┐
    │  ACTION  │ ← What's the simplest behavior in anticipation of reward?
    └────┬─────┘
         ▼
    ┌──────────┐
    │ VARIABLE │ ← What reward does the user get? (Must be variable!)
    │  REWARD  │
    └────┬─────┘
         ▼
    ┌──────────┐
    │INVESTMENT│ ← What does the user put in that improves the next cycle?
    └────┬─────┘
         │
         └──────────► Back to TRIGGER
```

**Key insight:** Habits are NOT formed by making products "sticky" or "addictive." They're formed by repeatedly cycling users through this loop until the behavior becomes automatic.

## Phase 1: Trigger

The trigger is what initiates the behavior. Two types:

### External Triggers (Start the cycle)

| Type | Example | Purpose |
|------|---------|---------|
| **Paid** | Ads, SEM, sponsorships | Acquire new users |
| **Earned** | PR, viral content, app store | Grow through attention |
| **Relationship** | Referrals, word of mouth, invites | Grow through trust |
| **Owned** | Push notifications, email, app icon | Re-engage existing users |

External triggers get users INTO the loop. But the goal is to transition to...

### Internal Triggers (Sustain the cycle)

Internal triggers are **emotions** that prompt product use:
- **Boredom** → Instagram, TikTok, Reddit
- **Loneliness** → Facebook, messaging apps
- **Uncertainty** → Google, Stack Overflow
- **Fear of missing out** → Twitter, news apps
- **Anxiety** → email checking, Slack

**The most powerful internal trigger:** A negative emotion the user wants to relieve.

**How to find your internal trigger:**
1. Ask "Why would someone use this?" 5 times
2. Map the emotion journey your user experiences
3. Look at WHEN users open your product (time of day, context)

## Phase 2: Action

The simplest behavior done in anticipation of reward. Defined by the Fogg Behavior Model:

```
Behavior = Motivation × Ability × Trigger
```

All three must be present simultaneously.

### Increasing Ability (Reduce friction)

Make the action as easy as possible:

| Friction Factor | How to Reduce |
|----------------|---------------|
| Time | Fewer steps, faster load |
| Money | Free tier, low entry price |
| Physical effort | Fewer taps/clicks |
| Brain cycles | Simpler UI, smart defaults |
| Social deviance | Show others doing it |
| Non-routine | Make it familiar |

**Rule of thumb:** Every extra step loses 20% of users. Obsess over removing steps.

**Examples of minimal actions:**
- Pinterest: Scroll (zero effort to see reward)
- Google: Type query (one action)
- Instagram: Open app (feed is already there)
- Slack: Glance at notification preview

## Phase 3: Variable Reward

The reward satisfies the user's need BUT with an element of variability. **Predictable rewards lose power. Variable rewards create craving.**

### Three Types of Variable Rewards

#### Rewards of the Tribe (Social)
- Social validation: likes, comments, followers
- Feeling of belonging: community activity, group chats
- Recognition: leaderboards, badges, mentions

**Examples:** Facebook likes, Stack Overflow reputation, Reddit upvotes

#### Rewards of the Hunt (Resources/Information)
- New information: news feeds, email inbox
- Resources: deals, job opportunities, leads
- Money: variable payouts, commission structures

**Examples:** Twitter feed, LinkedIn jobs, Slot machines

#### Rewards of the Self (Mastery/Completion)
- Completion: clearing notifications, finishing levels
- Mastery: skill improvement, streaks
- Competence: solving problems, learning

**Examples:** Codecademy progress, Duolingo streaks, inbox zero

### Designing Variable Rewards

- Combine 2-3 reward types for stronger hooks
- **Autonomy is critical** — User must feel in control, not manipulated
- Variability keeps it interesting (same reward every time = boring)
- Finite variability (limited content) < Infinite variability (user-generated content)

## Phase 4: Investment

The user puts something into the product that:
1. **Increases the value** of the product for next use
2. **Increases the switching cost** (makes leaving harder)
3. **Loads the next trigger** (sets up the next cycle)

### Types of Investment

| Investment Type | Example | How It Improves Product |
|-----------------|---------|------------------------|
| **Data** | Profile info, preferences | Better personalization |
| **Content** | Posts, photos, documents | More to lose if you leave |
| **Followers** | Social connections, audience | Network effect, reach |
| **Reputation** | Reviews, karma, ratings | Status, trust |
| **Skill** | Learning the interface, shortcuts | Proficiency = speed |
| **Effort** | Customizations, configurations | Tailored experience |

**Timing:** Ask for investment AFTER the reward, when the user is satisfied. Never before.

**Investment loads the next trigger:**
- Adding friends → friend activity triggers return
- Setting preferences → better recommendations trigger return
- Creating content → comments/likes trigger return

## The Manipulation Matrix

Ethical framework for habit-forming design:

|  | Maker Uses Own Product | Maker Does NOT Use Own Product |
|--|----------------------|-------------------------------|
| **Materially Improves Life** | **Facilitator** (Build this) | **Peddler** (Proceed with caution) |
| **Does NOT Improve Life** | **Entertainer** (Okay for some) | **Dealer** (Don't build this) |

**The two key questions:**
1. Would I use this product myself?
2. Does this product materially improve the user's life?

If both answers are "yes" → you're a Facilitator. Build with confidence.

## Measuring Habit Strength

| Metric | What It Tells You | Target |
|--------|-------------------|--------|
| DAU/MAU ratio | What % of monthly users come daily | >20% (good), >50% (strong habit) |
| Session frequency | How often users open the product | Depends on use case |
| Time to habit | How many cycles before automatic use | Track per cohort |
| Retention curve | Does usage flatten or drop to zero? | Must flatten |
| Unprompted return | Users who return without notification | Higher = stronger habit |

## Common Mistakes

1. **No variable reward** — Same experience every time = no anticipation = no habit
2. **Investment before reward** — Asking users to put in effort before they've gotten value
3. **External triggers only** — If users only come from notifications, you don't have a habit
4. **Ignoring the ethics** — Manipulative patterns destroy trust and invite regulation
5. **Too many actions required** — If the action isn't simple, users won't do it repeatedly
