# Product Metrics Dashboard

## AARRR (Pirate Metrics)

| Stage | Key Metric | How to Measure |
|-------|-----------|----------------|
| **Acquisition** | New signups, traffic by source | Google Analytics, signup events |
| **Activation** | Users reaching "aha moment" | Custom event tracking |
| **Retention** | DAU/MAU, Day 1/7/30 retention | Cohort analysis |
| **Revenue** | Conversion to paid, MRR, ARPU | Payment events, billing system |
| **Referral** | NPS, viral coefficient, referral signups | Surveys, referral tracking |

## Product-Market Fit Signals

### Sean Ellis Survey
Ask users: "How would you feel if you could no longer use [product]?"
- Very disappointed: >40% = PMF achieved
- Somewhat disappointed: 25-40% = Getting close
- Not disappointed: <25% = PMF not reached

### Retention Curve
- Good: Curve flattens (users stick around)
- Bad: Curve declines to zero (everyone leaves eventually)
- Great: Curve goes UP (users increase usage over time)

### Organic Growth
- Word of mouth > paid growth
- Users coming back without prompting
- Unsolicited positive feedback
- Users requesting more features (not just complaining)

## Key Metrics by Product Type

### SaaS
| Metric | Good | Great |
|--------|------|-------|
| Trial → Paid conversion | 10-15% | >25% |
| Monthly churn | <5% | <2% |
| NPS | >30 | >50 |
| Activation rate | >25% | >50% |
| DAU/MAU | >20% | >40% |

### B2C App
| Metric | Good | Great |
|--------|------|-------|
| Day 1 retention | >25% | >40% |
| Day 7 retention | >10% | >20% |
| Day 30 retention | >5% | >10% |
| Session frequency | 3x/week | Daily |
| Viral coefficient | >0.5 | >1.0 |

### Marketplace
| Metric | Good | Great |
|--------|------|-------|
| Liquidity (match rate) | >30% | >60% |
| Take rate | 10-20% | 15-25% |
| Repeat transaction rate | >30% | >50% |
| Supply/demand balance | Slight surplus | Balanced |

## Dashboard Design Principles
1. **Lead with one North Star metric** — the one number that matters most
2. **Show trends, not snapshots** — 4-week or 12-week view
3. **Compare to targets** — actual vs goal
4. **Segment by cohort** — don't blend, it hides problems
5. **Include leading indicators** — metrics that predict future outcomes
6. **Update automatically** — manual dashboards get stale
